This is a dump of the names of records in the property table and gives an indication
of how the data is organized.

Each window section begins with a record named <windowclassname>PARTS
The section for that window class then continues until the next record ending in the string PARTS

Immediately following the PARTS record are 1...n records denoting the parts of that window class.
The parts list continues until the first record that contains STATES or PARTS

Following the PARTS list will either be a new window class section or 1..n part states records.
Part state records begin with <partname>STATES and continue until the next record that contains PARTS or STATES.
In the states sction may also be a set of states for the window class itself, so there may be a record
named <windowclassname>STATES

The UxTheme dll does not seem to like certain records in the table and returns E_FAIL if you try
and draw using them. One example is the WINDOWPARTS::SYSBUTTON parts.

There are also some records for which OpenThemeData returns NULL. CLOCKPARTS is an example of this.
